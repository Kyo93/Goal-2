# Proposal: IT Operations Knowledge Package for Alpha Intelligence

Generated by: `cm-brainstorm-idea` and `cm-planning`  
Date: 2026-06-02

## Why We're Doing This

IT Operations has three raw Excel exports with complementary evidence:

- `IssueReport.xlsx`: user-raised tickets and comments.
- `SEA - Corp IT- ILL- Incident Report   (Responses).xlsx`: confirmed incidents, troubleshooting actions, and RCA.
- `zbx_problems_export.xlsx`: raw Zabbix alerts.

The goal is not to build another dashboard or chat application. The goal is to convert these files into a reliable knowledge package that Alpha Intelligence can use to answer operational questions.

The primary product question is timeline-first:

> During a selected time range, what happened in the IT system, what did Zabbix
> detect, was there evidence of user impact, and was the confirmed cause
> attributable to an internal system, an ISP, an external dependency, or still
> unknown?

Typical investigation questions:

- What confirmed incidents occurred during a selected time range?
- What Zabbix signals occurred before, during, or after those incidents?
- Which sites, services, or users were affected?
- Was user impact confirmed, estimated, unsupported, or unknown?
- What RCA was recorded and how is the responsibility domain classified?
- Was the issue resolved permanently?
- Which patterns recur?

## Qualified Problem

**For:** IT Operations, NOC, and IT managers  
**Who:** want to investigate operational history through Alpha Intelligence  
**The:** IT Operations knowledge package is a normalized, evidence-backed set of documents and audit tables  
**That:** allows an Alpha Super Agent to answer flexibly without calculating statistics from raw spreadsheets  
**Unlike:** uploading raw Excel files directly into RAG  
**Our approach:** uses a deterministic Python converter outside Alpha, then uploads curated Markdown documents into a Knowledge Expert used by a Super Agent

### Root Causes

1. Alerts, tickets, and confirmed incidents have different meanings and schemas.
2. Monitoring alerts contain repeated noise and cannot be counted directly as incidents.
3. RAG is suitable for retrieving curated knowledge chunks, but it should not be trusted to count thousands of spreadsheet rows reliably.
4. Host-to-site, service, and user-group master data is incomplete.

## Options Evaluated

### Option A: Upload Raw Files Directly Into RAG

**Pros:** minimal setup.  
**Cons:** weak counting accuracy, weak deduplication, and high risk that alerts are misrepresented as incidents.

### Option B: Python Converter + Knowledge Expert + Super Agent

Run a deterministic Python converter outside Alpha, upload curated Markdown into a Knowledge Expert, then attach that expert to a Super Agent.

**Pros:** traceable, testable, portable, easy to audit, and suited to flexible investigation questions.  
**Cons:** requires a conversion step whenever raw exports change.

### Option C: Build Processing Entirely Inside Alpha Workflow

Use Alpha Workflow file inputs and Code Executor nodes to normalize every export.

**Pros:** one-platform user experience.  
**Cons:** Code Executor has a five-second timeout; large spreadsheet processing and validation become fragile.

## Recommendation

Choose **Option B: Python Converter + Knowledge Expert + Super Agent**.

Alpha Intelligence official guides describe:

- Workflows as suitable for data and file processing.
- Knowledge Experts as RAG-powered knowledge bases.
- Super Agents as autonomous orchestrators that dynamically select tools and knowledge.
- Code Executor as supporting Python, `pandas`, and `openpyxl`, but limited to five seconds per node.

The converter should therefore run outside Alpha. Alpha should focus on retrieval and reasoning.

## Target Flow

```text
3 raw Excel files
  + optional reviewed master data
  -> Python converter
  -> curated Markdown knowledge package + normalized_data.xlsx
  -> manual upload to Alpha Knowledge Expert for PoC
  -> Alpha Super Agent with the Expert attached
  -> flexible evidence-backed Q&A
```

## Scope

### In Scope

- Python converter executed locally outside Alpha Intelligence.
- Validation of the expected schemas for all three raw files.
- Timestamp normalization and deterministic deduplication.
- Ticket comment aggregation by ticket ID.
- Confirmed-incident normalization.
- Zabbix alert signature normalization and alert-pattern grouping.
- Deterministic recurrence summaries.
- Deterministic operational timeline records ordered by timestamp.
- Conservative timeline correlation between confirmed incidents, Zabbix patterns, and direct ticket evidence.
- Responsibility-domain classification as `INHOUSE`, `ISP`, `EXTERNAL`, or `UNKNOWN`.
- Data-quality findings and explicit unknown values.
- Markdown package optimized for RAG chunk retrieval.
- `normalized_data.xlsx` for human audit.
- Machine-readable and human-readable validation reports.
- Setup guide for Alpha Knowledge Expert and Super Agent.
- Manual package upload for the PoC.

### Out of Scope

- Local dashboard, database, or custom chat application.
- Processing the spreadsheets inside Alpha Code Executor.
- Automatic Knowledge Expert synchronization.
- Realtime source-system integrations.
- Automated incident creation, mutation, closure, or escalation.
- Autonomous RCA conclusions without evidence.
- Treating unreviewed master-data mappings as facts.

## Guardrails

- Raw alert is never presented as confirmed incident.
- Timeline correlation is evidence linkage, not proof of RCA or user impact.
- Statistics come from the converter, not from RAG counting.
- Confirmed RCA and AI hypothesis use different labels.
- Responsibility classification derived by converter rules is labeled as a computed classification.
- Missing values are reported as `UNKNOWN`.
- Estimated impact is labeled explicitly.
- Each Markdown record contains stable evidence IDs and source lineage.
- Master-data rows are authoritative only when `review_status=CONFIRMED`.
- A generated package must not be uploaded to Alpha unless its validation report is `PASS`.

## Deliverables

- Python converter and focused tests.
- Knowledge package specification.
- Generated Alpha-ready package from the supplied sample files.
- Validation report with reconciliation evidence and upload-gate result.
- Alpha Knowledge Expert upload guide.
- Super Agent instruction template with preset investigation questions.
