ISP Incident Report Data Cleaning Logic

Tài liệu mô tả cách xử lý ISP Incident Report trong Alpha Intelligent: từ raw Google Sheet/CSV/XLSX thành confirmed incident records, Markdown file 02_confirmed_incidents.md, và JSON sạch để connector/query workflow dùng về sau.

1. Vai trò của ISP Incident Report

ISP Incident Report là source of truth cho confirmed incident. Nếu một sự kiện nằm trong ISP Incident Report hợp lệ, nó được coi là CONFIRMED_INCIDENT. Zabbix alert và ITcenter ticket chỉ bổ sung context hoặc user evidence, không tự thay thế nguồn xác nhận này.

Confirmed incident

Nguồn chính để xác định incident thật, thời điểm bắt đầu, thời điểm khôi phục, loại lỗi và RCA.

Evidence priority

Khi thiếu/mâu thuẫn dữ liệu, ưu tiên ISP Incident Report, sau đó ITcenter Ticket, rồi Zabbix Alert.

Knowledge-ready

Output Markdown dùng để upload Knowledge, còn JSON dùng cho connector và query workflow.

2. Luồng xử lý

1. Raw source Google Sheet CSV, uploaded CSV, hoặc XLSX form response.
2. Header mapping Chuẩn hóa header và map alias về schema chuẩn.
3. Row validation Parse date/time, site, RCA, severity; reject row lỗi datetime.
4. Normalize Sinh CONFIRMED_INCIDENT records với evidence/source_ref.
5. Render output Trả JSON, Markdown, và file payload 02_confirmed_incidents.md.

3. Input contract

Alpha node nhận một trong hai dạng input:

Input Dạng dữ liệu Khi dùng
incident_report_body CSV body từ HTTP/Google Sheet export Khi Alpha tải Google Sheet dưới dạng CSV.
incident_report_file CSV/XLSX uploaded file hoặc file path Khi user upload file vào Alpha workflow.
Nếu HTTP body là HTML/login page, processor phải trả lỗi rõ: HTTP body looks like HTML/login page; expected CSV export. Không được xử lý HTML thành incident.

Các cột sau là schema bắt buộc của incident report:

Column Vai trò

ISPProvider liên quan, dùng trong recurrence key và context.
SITE / LocationParse thành site_code và site_name.
Incident DATE, Incident TIMEGhép thành started_at.
ReporterNgười báo cáo hoặc form reporter.
Incindent TypeLoại incident. Giữ cả typo gốc để match form hiện tại.

5. Header alias logic

Processor dùng hàm normalize header: lower-case, trim, bỏ ký tự không phải chữ/số. Nhờ vậy các biến thể như site/location, site / location, incident date vẫn được map về cột chuẩn. 

_normalize_header("SITE / Location") -> "sitelocation"
HEADER_ALIASES["sitelocation"] -> "SITE / Location"
Form hiện tại có typo Incindent Type. Processor cố ý giữ alias này để không làm hỏng compatibility với Google Sheet đang chạy.

6. Row validation và reject rules

Trường hợp Xử lý Lý do
Blank row Tăng skipped_blank_rows Không phải incident.
Missing header Return error No header row found Không thể map schema.
Missing required columns Return error với danh sách cột thiếu Không xử lý thiếu schema bắt buộc.
Invalid incident date/time Append vào rejected_rows Không có started_at thì không query timeline được.
Invalid resolution date/time Append vào rejected_rows Tránh tạo duration/resolution sai.
Minimal content all blank/UNKNOWN Skip row Tránh tạo incident rỗng.

7. Normalized incident record

Mỗi row hợp lệ tạo ra một record CONFIRMED_INCIDENT.

{
  "record_type": "CONFIRMED_INCIDENT",
  "incident_id": "INC-3",
  "source_ref": "google_sheet_csv:row-3",
  "submitted_at": "2025-12-03T20:16:57",
  "started_at": "2025-12-03T19:18:00",
  "resolved_at": "2025-12-03T19:26:00",
  "duration_minutes": 8,
  "site_code": "MS2",
  "site_name": "Mega SOC - Cu Chi",
  "isp": "VNPT",
  "reporter": "Dang Quoc Binh",
  "incident_type": "High latency",
  "severity": "Minor",
  "resolution_status": "RESOLVED",
  "responsibility_domain": "ISP",
  "responsibility_basis": "RCA keyword rule: routing",
  "evidence_label": "SOURCE FACT",
  "evidence_note": "Incident row 3"
}
evidence_label của raw incident row luôn là SOURCE FACT. Các field tính toán như duration_minutes hoặc responsibility_domain cần được hiểu là computed từ source row, không phải user/ticket confirmation.

8. Field transformation rules

Output field Logic Ghi chú
incident_id INC-{row_idx} Ổn định theo row number trong sheet/export.
source_ref google_sheet_csv:row-{row_idx} hoặc {file}:{sheet}#row:{row_idx} Dùng để cite nguồn.
started_at Ghép Incident DATE + Incident TIME Bắt buộc.
resolved_at Ghép resolution date/time nếu có Không có thì resolution_status = UNKNOWN.
duration_minutes Floor số phút giữa resolved_at và started_at, không âm Ví dụ 8 phút.
site_code, site_name Split từ SITE / Location Hỗ trợ format MS2 - Mega SOC, MS2 Mega SOC.
description Incident description + reporter + submitted timestamp Initial Cause không merge vào description nữa, giữ field riêng.
recurrence_key site_code|isp|incident_type lower-case Dùng để group recurrence sau này.

9. Responsibility domain classification

responsibility_domain được tính từ Root Cause bằng keyword rule. Đây là phân loại hỗ trợ điều tra, không thay thế RCA gốc.

Domain Keyword examples Ví dụ
ISP routing, uplink, fiber, cable, provider, vnpt, fpt, cmc International routing fluctuations, fiber cable issue.
INHOUSE local segment, internal, switch, firewall, configuration, wifi controller Internal switch/firewall/config issue.
EXTERNAL electricity, utility, power supply, third-party Power/utility related outage.
UNKNOWN Không có keyword hoặc RCA blank/unknown Không đủ dữ liệu để classify.

10. Output contract

Machine-readable output

  • json_text: JSON string cho SuperAgent/tool chaining.
  • json_result: object chứa records/rejected rows.
  • summary: count và upload readiness.

Knowledge-ready output

  • markdown: nội dung Markdown.
  • confirmed_incidents_file: file-like payload tên 02_confirmed_incidents.md.
  • Không trả path string thay cho file content.

Summary fields

{
  "ok": true,
  "input_rows": 54,
  "record_count": 54,
  "rejected_row_count": 0,
  "skipped_blank_rows": 1,
  "reconciliation_passed": true,
  "upload_ready": true
}

11. Markdown file safety

Alpha từng gặp lỗi khi trả dict/path string làm file upload. Vì vậy processor phải trả BytesIO file-like object cho confirmed_incidents_file.

Safety rule Lý do
Write temp file 02_confirmed_incidents.md Traceability và verify file size.
Return io.BytesIO(md_bytes) Đảm bảo upload node nhận đúng content, không nhận wrapper text nhỏ.
Set bio.name = "02_confirmed_incidents.md" Giúp uploader nhận đúng filename.
Validate payload byte length Chặn lỗi silent tiny/missing Markdown file.
Không thay đổi phần _make_markdown_file() nếu không test lại upload Knowledge. Đây là đoạn bảo vệ quan trọng nhất cho file 02_confirmed_incidents.md.

13. Validation checklist

Check Expected
Google Sheet/CSV parse được summary.ok = true
Record count baseline Hiện tại Google Sheet mới: record_count = 54, rejected_row_count = 0.
Required downstream fields Có record_type, duration_minutes, responsibility_domain, evidence_label, source_ref.
Evidence label evidence_label = SOURCE FACT, evidence_note = Incident row N.
Markdown upload file confirmed_incidents_file.name = 02_confirmed_incidents.md và payload byte length bằng Markdown byte length.
Knowledge upload File lưu lên Knowledge không bị tiny wrapper, mở ra thấy incident records đầy đủ.

14. Cách SuperAgent nên hiểu ISP Incident Report

Claim Được phép nói Không được nói
Incident existence Có confirmed incident nếu nằm trong ISP Incident Report hợp lệ. Không cần Zabbix để xác nhận incident đã tồn tại.
RCA Dùng root_cause từ incident form và cite source_ref. Không tự tạo RCA mới từ Zabbix/ticket.
User impact Chỉ nói confirmed nếu có ticket/user evidence hoặc field explicit. Không suy user impact chỉ từ incident type.
Responsibility Nói là computed classification từ RCA keyword rule. Không coi là hợp đồng/SLA attribution nếu RCA không đủ rõ.
